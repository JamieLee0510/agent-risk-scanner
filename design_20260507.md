應該會有三個層級：

- Output Scanner，只能看到Output
- Tool-call Scanner，可以看到agent準備使用的tool
- Trace / Span Scanner，觀察完整的agent loop

### Output Scanner
只能掃以下結果：

- final answer
- file diff
- command output
- sandbox changes

進而檢查

- 是否洩漏 secret / token
- 是否產生惡意指令
- 是否產生違反 policy 的內容
- 是否修改了不該修改的檔案
- 是否輸出 suspicious URL / exfiltration target

### Tool-call Scanner
可以掃到

```
User intent
+
Untrusted context
+
Proposed tool call
+
Tool arguments
```

然後查看Agent 的決策：allow / ask_confirmation / block


### Trace / Span Scanner
需要像 OTEL 一樣做wrapper，攔截request/response，獲取：

```
agent.start
llm.request
llm.response
retriever.query
retriever.result
tool.call.proposed
tool.call.executed
tool.result
agent.final_response
```

每個span都可以有meta data：

```
{
  "span_type": "tool.call.proposed",
  "tool_name": "send_email",
  "arguments": {
    "to": "attacker@example.com"
  },
  "source": "llm_planner",
  "caused_by": "retrieved_content",
  "user_intent": "summarize document"
}
```

進而做更高階的判斷，如：

- agent 是否偏離 user intent
- tool call 是否由 untrusted content 觸發
- 是否發生 suspicious action chain
- retrieved document 是否污染後續 decision
- tool result 是否包含 instruction injection
- 是否有 privilege escalation
- 是否有 secret exfiltration

#### Memory Attacks（評估缺口最大的子家族）

> 2025 Q4 - 2026 Q2 之間新生成的攻擊面。Agent 把 RAG 用於長期記憶時，KB 變雙向、跨 session、自我增強——污染會跨會話殘留（"persistent behavioral drift"）。**目前無統一 benchmark**，最常被沿用的是 SafeRAG。

| Benchmark | 測什麼資安 | URL |
|---|---|---|
| **MemoryGraft**（8H6WCWWE）| 黑盒間接攻擊：透過 benign session 把「成功 procedure templates」植入 long-term memory | https://github.com/Jacobhhy/Agent-Memory-Poisoning |
| **SafeRAG**（被 XAMT 沿用）| 較早的 RAG 安全 benchmark；多篇 memory attack 沿用其 setup | https://github.com/IAAR-Shanghai/SafeRAG |
